好的，以下是基于之前功能模块的**详细交互流程设计**，按用户角色（普通用户、信息发布者、后台管理员）和业务场景分别描述，不包含代码。

## 一、用户端核心交互流程

### 1. 招聘求职流程

#### 场景A：求职者找全职/兼职/临时工

1. 用户进入小程序首页，点击「招聘」图标或顶部导航栏。
2. 进入招聘列表页，默认按发布时间倒序显示。
   - 顶部有筛选栏：可选择「全职」「兼职」「临时工」类型；
   - 可进一步筛选：薪资范围、距离（需授权位置）、是否包食宿。
3. 用户点击感兴趣的一条职位卡片 → 进入职位详情页：
   - 看到：公司名称、岗位职责、薪资待遇、工作地点、联系人电话/微信、是否包食宿、发布时间。
   - 底部固定两个按钮：「打电话」「在线聊」（如果开启私信）。
4. 用户点击「打电话」→ 调起手机拨号盘，直接拨打预留电话（号码部分掩码，需用户确认拨打）。
5. 用户点击「在线聊」→ 跳转到私信聊天界面，可向发布者发送文字/图片询问。
6. 用户想保存职位 → 点击右上角「收藏」，在「个人中心-我的收藏」可查看。
7. 用户投递简历（如果需要）：
   - 点击「投递简历」→ 系统检测用户是否已创建简历（姓名、年龄、工作经验等）。
   - 若未创建，引导填写基础简历（一次性）。
   - 提交后，发布者收到系统通知“有用户投递了您的职位”。
8. 求职者可在「个人中心-我的求职」查看投递记录和面试邀请（发布者可在聊天窗口发送邀请）。

#### 场景B：企业/个人发布招聘信息

1. 用户点击首页「发布」按钮（或底部导航「我的」→「发布招聘」）。
2. 选择发布类型：「全职」「兼职」「临时工」。
3. 填写表单：职位名称、薪资待遇（元/天或元/月）、工作地点、详细描述、联系人、联系电话（是否公开）、是否包食宿、上传图片（可选）。
4. 系统要求实名认证（首次发布）：
   - 个人发布需上传身份证照片+手机验证；
   - 企业发布需上传营业执照+管理员身份证（后台审核）。
5. 提交后，状态为「待审核」，用户可在「我的发布」中查看进度。
6. 审核通过 → 信息在列表展示；审核不通过 → 收到系统通知，说明拒绝原因，可修改后重提。

### 2. 租房二手房流程

#### 场景A：租客/购房者浏览房源

1. 首页点击「租房二手房」入口。
2. 进入房源列表页，顶部标签切换：「出租」「出售」「商铺转让」。
3. 筛选：可点「筛选」按钮，选择价格区间、户型（1室/2室等）、面积、区域（舞阳县下属乡镇或街道）。
4. 点击房源卡片 → 详情页展示：
   - 图片轮播、标题、价格、户型、面积、装修情况、楼层、地址、联系人、电话。
   - 地图查看位置（可一键导航）。
   - 底部按钮：「打电话」「在线聊」「收藏」。
5. 用户可联系房东/中介（电话或聊天）。
6. 如果房源已租/已售，用户可点击「举报」告知平台下架。

#### 场景B：发布房源（个人或中介）

1. 点击「发布房源」。
2. 选择发布类型：「出租」「出售」「商铺转让」。
3. 填写表单：
   - 标题（如“舞阳县中心三室两厅出租”）
   - 价格（月租/总价）
   - 户型、面积、楼层、装修情况
   - 地址（支持地图选点或手动输入）
   - 描述（配套设施、要求等）
   - 上传图片（最多9张）
   - 联系人、联系电话（可设置“仅展示微信”避开电话骚扰）
4. 身份验证：
   - 个人发布：手机号实名 + 身份证号（后台留痕）。
   - 中介发布：需上传营业执照认证（首次需审核）。
5. 提交审核 → 后台审核（1小时内，人工或自动关键词过滤）。
6. 审核通过后，房源上架；用户可在线下架或修改（修改需重审）。

### 3. 便民信息流程（拼车/失物招领/打听事/便民求助）

#### 场景A：用户查看或搜索便民信息

1. 底部导航点击「便民」。
2. 顶部显示四个分类标签：「拼车」「失物招领」「打听事」「便民求助」，默认选中「拼车」。
3. 每个分类下列表展示：
   - 拼车：出发地→目的地、时间、人数、价格（人找车/车找人）。
   - 失物招领：标题、物品名称、丢失/拾到地点、日期。
   - 打听事：问题标题、描述、悬赏（如有）。
   - 便民求助：服务类型（开锁、维修等）、需求描述、期望时间。
4. 用户可点击详情查看完整描述和联系电话，直接拨打联系。

#### 场景B：发布便民信息

1. 点击「发布」按钮（悬浮或右上角）。
2. 选择信息类型（拼车/失物招领/打听事/便民求助）。
3. 填写对应字段（例如拼车需要填写车型、座位数、是否允许带宠物等；失物招领需要上传物品图片）。
4. 填写联系方式（手机号、微信），可勾选“隐藏号码仅通过私信联系”。
5. 提交前需手机验证码确认（防刷量）。
6. 提交后立即显示（或经后台自动审核/人工审核，视平台策略而定）。
7. 发布者可在「我的发布」中下架信息（例如拼车已满员、已找到失物）。

#### 场景C：响应便民信息（例如看到失物招领想联系）

1. 用户点击信息详情页。
2. 点击「联系TA」→ 若发布者开放电话则直接拨打；若仅开放私信则跳转到聊天。
3. 对于「打听事」，其他用户可在详情页下方「回答」输入文字，发布者会收到通知。
4. 发布者可对回答进行采纳（如有悬赏积分）。

### 4. 商家服务流程

#### 场景A：用户查找商家并联系

1. 首页点击「服务」入口。
2. 进入商家分类页：显示分类图标（餐饮、家政、装修等），每个分类下显示热门商家列表。
3. 用户点击某个分类 → 进入该分类下的商家列表（支持按距离排序，需授权位置）。
4. 点击商家卡片 → 商家主页：
   - 名称、地址、营业时间、联系电话、相册、优惠活动（如有）、用户评价（如果有）。
   - 一键导航（打开地图App）。
   - 收藏商家。
5. 用户可打电话预约服务或咨询。

#### 场景B：商家申请入驻

1. 商家在商家服务页点击「商家入驻」按钮。
2. 填写入驻表单：店铺名称、经营类目、联系人、联系电话、店铺地址、营业执照照片、店铺门头照。
3. 提交后，后台管理员审核。
4. 审核通过 → 商家获得入驻标识，可在后台管理自己的店铺信息（编辑相册、更新优惠、回复评价）。
5. 平台可支持付费套餐（基础版免费展示，付费版置顶+广告位）。

---

### 5. 二手闲置流程

#### 场景A：买家浏览与联系

1. 首页点击「二手闲置」。
2. 列表默认显示最新发布，可切换分类（手机/家电/家具/母婴/其他）。
3. 筛选：价格区间、成色（全新/几乎全新/有磨损）、区域。
4. 点击商品 → 详情页：图片、描述、原价/转让价、成色、交易地点（自提点）、卖家联系方式。
5. 买家通过「打电话」或「在线聊」与卖家沟通。
6. 线下交易完成后，买家可点击「举报」或“已售出”反馈（卖家也可标记已售）。

#### 场景B：卖家发布闲置

1. 点击「发布闲置」。
2. 拍照或从相册选择图片（最多9张）。
3. 填写标题、描述、价格（转让价）、原价（可选）、成色、交易地点（支持定位或手动输入）。
4. 设置联系方式（电话/微信），可选择“仅在线聊天不显示号码”。
5. 提交审核（系统自动过敏感词，或人工审核）。
6. 审核通过后上架。卖家可在「我的发布」中编辑、下架、标记“已售出”。

### 6. 本地资讯流程

- **用户侧**：
  1. 首页底部导航或中间区域展示「本地资讯」条目。
  2. 点击进入资讯列表，按时间倒序显示。
  3. 用户可点赞、分享（转发给微信好友或朋友圈）。
  4. 不支持用户自主发布资讯（仅管理员后台发布）。

- **管理员侧**（见后台流程）。

## 二、后台管理员审核流程

### 1. 信息审核（统一流程）

1. 管理员登录后台，进入「信息审核」模块。
2. 选择待审类型（招聘/房源/便民/二手/商家入驻）。
3. 列表展示每条待审信息的标题、发布时间、发布人、联系电话。
4. 点击「详情」查看完整内容和图片。
5. 两个操作：「通过」或「拒绝」。
   - 若拒绝，必须填写拒绝原因（如“图片违规”“联系电话无效”），系统将原因发送给用户。
6. 审核通过的信息自动在用户端展示。
7. 对于已发布的信息，管理员可随时「下架」或「置顶」（置顶需用户已付费或由运营操作）。

### 2. 用户认证审核（企业/中介/个人实名）

1. 后台「用户管理」→「认证申请列表」。
2. 查看用户提交的证明材料图片。
3. 审核通过后，该用户头像旁显示“认证企业”或“认证中介”标识；不通过则通知用户重新提交。

### 3. 广告位管理流程

1. 后台「广告管理」→ 添加广告。
2. 填写：广告名称、投放位置（首页轮播/信息流/详情页底部）、广告图片、跳转链接（外部或小程序页面）、开始时间/结束时间。
3. 提交后，广告按计划自动展示。管理员可随时下线。
4. 支持查看广告点击数据（PV/UV）。

### 4. 资讯发布流程（管理员）

1. 后台「资讯管理」→「新增资讯」。
2. 填写标题、正文（支持图文编辑）、来源、是否置顶、发布时间。
3. 上传封面图。
4. 发布后立即在前端资讯列表显示。管理员可编辑或删除。

### 5. 举报处理流程

1. 用户端举报 → 后台「举报管理」显示新举报，包括被举报信息ID、举报原因、举报人。
2. 管理员查看被举报内容，判断是否违规：
   - 违规 → 下架该信息，并给发布者发送警告或封禁；
   - 不违规 → 驳回举报，不处理信息。
3. 处理结果反馈给举报人（系统通知）。

## 三、通用交互细节（贯穿全流程）

### 1. 消息通知系统
- **触发节点**：
  - 信息审核通过/不通过 → 发送系统通知。
  - 用户收到咨询（在线聊天）→ 红点提示。
  - 用户收藏的内容有更新（如价格变动）→ 通知（可选）。
  - 举报结果反馈。
- **展示位置**：小程序底部导航「消息」中心，或微信服务通知（需模板消息授权）。

### 2. 发布信息时的防骚扰设计
- 用户可选择“隐藏手机号，仅通过在线聊天联系”，系统提供匿名聊天通道。
- 聊天中不可发送外部链接、二维码，系统自动拦截。

### 3. 信息排序与置顶规则
- 普通信息按发布时间倒序。
- 付费置顶信息（招聘/房源/二手）置顶显示，置顶到期后恢复自然排序。
- 置顶购买流程：用户点击“置顶”按钮 → 选择时长（1天/7天/30天）→ 支付（微信支付）→ 置顶生效。

### 4. 用户注销与数据删除
- 个人中心提供「注销账号」入口，用户确认后，后台删除其所有发布信息和聊天记录（或匿名化处理）。
