> 三端已完成联调（发布 → 审核 → 展示 → 收藏/投递/举报/消息通知闭环均验证通过）。

## 一、技术栈与端口

| 端 | 目录 | 技术栈 | 端口 |
|----|------|--------|------|
| 后端服务 | `server/` | NestJS 11 + Prisma 6 + SQLite | `3000`（接口前缀 `/api`，静态文件 `/uploads`） |
| 后台管理 | `admin/` | Vue3 + Element Plus + Vite | `5181`（代理 `/api`、`/uploads` → 3000） |
| 用户端 | `frontend/` | uni-app（Vue3）H5 / 微信小程序 | `5173`（代理 `/api`、`/uploads` → 3000） |

包管理器统一使用 **pnpm**（不要在仓库根目录使用 `npm install`，否则会把依赖提升到根
`node_modules`，导致 uni-app 解析到错误版本——uni-app 各包已锁定为
`3.0.0-alpha-5000920260512001`，新版本 alpha 存在 slot 渲染回归 bug）。

## 二、首次启动

```bash
# 1. 安装三端依赖（仓库根目录执行）
pnpm run install:all

# 2. 初始化数据库（建表 + 种子数据，仅首次需要；重置则先删除 server/prisma/dev.db）
pnpm run db:init

# 3. 分别启动三端（三个终端）
pnpm run dev:server     # http://127.0.0.1:3000/api
pnpm run dev:admin      # http://127.0.0.1:5181
pnpm run dev:frontend   # http://127.0.0.1:5173
```

## 四、后端结构

```
server/
├── prisma/
│   ├── schema.prisma          # 全部数据模型（内容/用户/RBAC/广告/订单/系统）
│   ├── seed.js                # 种子数据（角色权限、菜单、分类、示例内容、广告、配置）
│   └── cleanup-test-data.js   # 清理联调测试数据
├── src/
│   ├── auth/                  # 管理端 JWT 登录与鉴权守卫
│   ├── public/                # 用户端接口（/api/home、/api/listings、/api/profile/...）
│   ├── adminapi/              # 管理端接口（内容/用户/运营/系统/统计，前缀 /api/admin）
│   ├── uploads/               # 图片上传（/api/uploads，文件落盘 server/uploads）
│   └── common/                # Prisma、统一响应包裹 {data}、统一错误 {error}、工具
└── .env                       # DATABASE_URL / JWT_SECRET / PORT
```

约定（与两端请求层一致，勿改）：
- 成功响应一律包裹为 `{ "data": ... }`；错误为非 2xx + `{ "error": "中文消息" }`。
- 分页参数 `page` / `pageSize`，分页响应 `{ items, total, page, pageSize }`。
- 管理端资源名 ↔ 内容类型：`recruitments→jobs`、`houses→houses`、`conveniences→convenience`、
  `used-goods→secondhand`、`shops→yellowPages`、`articles→news`（一张 `Listing` 表统一存储）。
- 内容状态机：`pending → approved / rejected`，可 `offline`，删除为软删（`deleted`）。
- 审核/拒绝/下架/封禁会自动给发布者写入站内消息；管理端关键操作写入操作日志。

## 五、业务闭环（已验证）

1. 用户端发布（含敏感词检测：命中即强制送审或拒绝，可在后台「系统设置」切换策略）。
2. 进入管理端仪表盘「待审队列」，单条或批量通过/拒绝（拒绝必填原因）。
3. 通过后用户端列表/首页精选即时可见；用户消息中心收到审核通知。
4. 收藏、求职投递、举报（举报可在后台判违规并一键下架，双方均收到消息）。
5. 置顶（含优先级/到期时间，到期自动恢复）与推荐（首页精选位）。
6. 图片上传（`/api/uploads`，8MB 上限，jpg/png/gif/webp/svg）。

## 六、切换 MySQL（生产部署）

当前用 SQLite 便于本机零依赖联调；Dockerfile 按 MySQL 设计。切换步骤：

1. `server/prisma/schema.prisma` 中 `provider = "sqlite"` 改为 `"mysql"`；
2. `server/.env` 的 `DATABASE_URL` 改为 `mysql://user:password@host:3306/qingning_local_life`；
3. 删除 `server/prisma/migrations` 后执行 `npx prisma migrate dev --name init`（MySQL 方言重新生成迁移）；
4. `node prisma/seed.js` 重新灌种子数据。
   （schema 未使用 SQLite 特有能力，JSON 字段以字符串存储，直接兼容 MySQL。）
