# 广告位管理系统重构方案

**版本**: v2.0  
**更新日期**: 2026-06-11  
**状态**: 待实现

---

## 概述

当前广告系统设计存在逻辑混乱，与业界标准的数字广告管理体系不符。本方案通过重构，建立**广告位 → 计划 → 创意 → 投放 → 数据**的完整链路。

---

## 现状 vs 改进对比

### 当前设计的问题

```
问题1：概念混淆
├─ AdPosition（广告位）- pv/uv/ctr 存储在这里 ❌
├─ Ad（广告）- 只有 title/image/link，无投放管理
└─ 缺少：广告计划（Plan）、投放时间、预算等

问题2：管理流程不清晰
├─ 无法设置投放时间（仅有 startAt/endAt）
├─ 无法管理预算（无 budget 字段）
├─ 无法选择计费方式（无 pricingModel）
└─ 无法追踪投放效果（pv/uv/ctr 是静态种子值）

问题3：统计数据不合理
├─ pv/uv/ctr 在 AdPosition，不在具体广告
├─ 无法区分不同广告的效果
└─ 无法统计投放 ROI
```

### 改进后的设计

```
AdPositionCategory（广告位分类）
├─ 首页（Home）
│  ├─ 轮播图（Carousel）
│  └─ 精选推荐（Featured）
├─ 频道（Channel）
│  ├─ 招聘频道横幅
│  ├─ 房源频道横幅
│  └─ 便民频道横幅
├─ 详情页（Detail）
│  └─ 商品详情页推荐
└─ 列表页（List）
   └─ 列表页底部横幅

↓

AdPosition（广告位）
├─ 位置ID + 名称 + 分类
├─ 规格（宽×高、展示个数）
├─ 展示规则（轮播、随机、优先级）
└─ 容量限制（最多放 N 个广告）

↓

AdCampaign（广告计划）
├─ 计划名称 + 描述
├─ 投放期限（startDate - endDate）
├─ 投放位置（可多个）
├─ 投放预算 + 计费方式（CPM/CPC/CPA）
├─ 日限额
├─ 投放时段（如08:00-20:00）
└─ 状态（草稿/投放中/已下线）

↓

AdCreative（广告创意）
├─ 标题 + 描述
├─ 创意素材（图片/视频）
├─ 链接（分类/H5/小程序）
├─ 关联计划
└─ 状态（审核中/已通过/已拒绝）

↓

AdPerformance（投放数据）
├─ 曝光（Impression）
├─ 点击（Click）
├─ 点击率（CTR）
├─ 转化（Conversion，可选）
├─ 花费（Cost）
└─ ROI
```

## 数据模型设计

### 新增表结构

#### 1. AdPositionCategory（广告位分类）

```sql
CREATE TABLE ad_position_categories (
  id            VARCHAR(36) PRIMARY KEY,
  code          VARCHAR(32) UNIQUE NOT NULL,        -- home, channel, detail, list
  name          VARCHAR(64) NOT NULL,               -- "首页", "频道页"
  description   TEXT,
  icon          VARCHAR(255),
  sort_order    INT DEFAULT 0,
  status        VARCHAR(32) DEFAULT 'enabled',      -- enabled, disabled
  created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. AdPosition（广告位）- 改进版

```sql
CREATE TABLE ad_positions (
  id              VARCHAR(36) PRIMARY KEY,
  category_id     VARCHAR(36) NOT NULL,             -- 关联分类
  name            VARCHAR(96) NOT NULL,             -- "首页轮播图"
  description     TEXT,
  
  -- 规格信息
  width           INT,                              -- 像素宽度
  height          INT,                              -- 像素高度
  max_ads         INT DEFAULT 5,                    -- 最多放几个广告
  display_rule    VARCHAR(32) DEFAULT 'carousel',   -- carousel, random, priority, fixed
  
  -- 展示信息
  scene           VARCHAR(64),                      -- 场景标识（兼容原字段）
  min_impression  INT DEFAULT 0,                    -- 最小曝光要求
  
  status          VARCHAR(32) DEFAULT 'enabled',
  created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  
  FOREIGN KEY (category_id) REFERENCES ad_position_categories(id),
  INDEX(category_id, status)
);
```

#### 3. AdCampaign（广告计划） - 新建

```sql
CREATE TABLE ad_campaigns (
  id              VARCHAR(36) PRIMARY KEY,
  name            VARCHAR(128) NOT NULL,            -- 计划名称
  description     TEXT,
  
  -- 投放期限
  start_date      DATE NOT NULL,
  end_date        DATE NOT NULL,
  
  -- 预算与计费
  budget          DECIMAL(12, 2),                   -- 总预算
  daily_budget    DECIMAL(12, 2),                   -- 日限额
  pricing_model   VARCHAR(32) DEFAULT 'cpm',        -- cpm(按千次展示), cpc(按点击), cpa(按转化)
  unit_price      DECIMAL(10, 4),                   -- 单价（元/千次、元/次等）
  
  -- 投放设置
  position_ids    JSON,                             -- 投放位置数组
  time_slots      JSON,                             -- 投放时段 [{start: "08:00", end: "20:00"}]
  
  -- 广告主信息
  advertiser_id   VARCHAR(36),                      -- 广告主ID（可选）
  advertiser_name VARCHAR(96),
  contact_phone   VARCHAR(32),
  contact_email   VARCHAR(128),
  
  -- 状态
  status          VARCHAR(32) DEFAULT 'draft',      -- draft, pending, active, paused, ended, offline
  reject_reason   TEXT,                             -- 拒绝原因
  
  -- 统计（实时）
  total_impression  BIGINT DEFAULT 0,
  total_click       BIGINT DEFAULT 0,
  total_cost        DECIMAL(12, 2) DEFAULT 0,
  
  created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  started_at      TIMESTAMP,
  ended_at        TIMESTAMP,
  
  INDEX(status, start_date),
  INDEX(created_at)
);
```

#### 4. AdCreative（广告创意） - 重构版

```sql
CREATE TABLE ad_creatives (
  id              VARCHAR(36) PRIMARY KEY,
  campaign_id     VARCHAR(36) NOT NULL,             -- 关联计划
  title           VARCHAR(128) NOT NULL,
  description     TEXT,
  
  -- 素材
  image_url       VARCHAR(255),                     -- 主图
  thumbnail_url   VARCHAR(255),                     -- 缩略图
  video_url       VARCHAR(255),                     -- 视频（可选）
  
  -- 链接
  link_type       VARCHAR(32) DEFAULT 'category',   -- category, h5, miniapp, external
  link_value      VARCHAR(255),                     -- 分类code / URL / 小程序路径
  
  -- 审核
  status          VARCHAR(32) DEFAULT 'pending',    -- pending, approved, rejected
  reject_reason   TEXT,
  reviewed_by     VARCHAR(64),                      -- 审核人
  reviewed_at     TIMESTAMP,
  
  -- 统计
  impression      BIGINT DEFAULT 0,
  click           BIGINT DEFAULT 0,
  conversion      BIGINT DEFAULT 0,
  
  created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  
  FOREIGN KEY (campaign_id) REFERENCES ad_campaigns(id) ON DELETE CASCADE,
  INDEX(campaign_id, status),
  INDEX(status)
);
```

#### 5. AdPerformance（投放数据） - 新建

```sql
CREATE TABLE ad_performance (
  id              VARCHAR(36) PRIMARY KEY,
  creative_id     VARCHAR(36) NOT NULL,
  position_id     VARCHAR(36) NOT NULL,
  campaign_id     VARCHAR(36) NOT NULL,
  
  -- 时间维度（按天统计）
  stat_date       DATE NOT NULL,
  
  -- 曝光数据
  impression      BIGINT DEFAULT 0,
  unique_user     BIGINT DEFAULT 0,
  
  -- 点击数据
  click           BIGINT DEFAULT 0,
  click_rate      DECIMAL(5, 2) DEFAULT 0,         -- CTR %
  
  -- 转化数据（可选）
  conversion      BIGINT DEFAULT 0,
  conversion_rate DECIMAL(5, 2) DEFAULT 0,         -- CVR %
  
  -- 花费数据
  cost            DECIMAL(12, 2) DEFAULT 0,
  cpm             DECIMAL(10, 4) DEFAULT 0,        -- 千次展示成本
  cpc             DECIMAL(10, 4) DEFAULT 0,        -- 单次点击成本
  
  created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  
  FOREIGN KEY (creative_id) REFERENCES ad_creatives(id),
  FOREIGN KEY (position_id) REFERENCES ad_positions(id),
  FOREIGN KEY (campaign_id) REFERENCES ad_campaigns(id),
  UNIQUE KEY (creative_id, position_id, stat_date),
  INDEX(stat_date),
  INDEX(campaign_id)
);
```

## 管理流程优化

### 新的管理流程

```
【第一步】广告位设置
├─ 创建分类（首页、频道、详情页等）
├─ 为每个分类创建具体的位置
│  └─ 设置规格、展示规则、容量
└─ 位置状态管理（启用/禁用）

【第二步】创建广告计划
├─ 填写基本信息
│  ├─ 计划名称、描述
│  ├─ 投放期限（从 XX 到 XX）
│  └─ 广告主信息
├─ 设置投放配置
│  ├─ 选择投放位置（支持多个）
│  ├─ 设置投放时段（如 8:00-20:00）
│  └─ 设置日限额
├─ 设置计费方式
│  ├─ 选择模式（CPM/CPC/CPA）
│  └─ 输入单价
└─ 提交审核或直接激活

【第三步】上传广告创意
├─ 为计划上传创意素材
│  ├─ 上传图片、标题、描述
│  ├─ 设置链接（分类/H5/小程序）
│  └─ 输入文案
├─ 创意审核（内容、链接、合规）
└─ 审核通过后可投放

【第四步】投放管理
├─ 查看计划状态
│  ├─ 投放中、已暂停、已下线
│  └─ 实时投放数据
├─ 动态调整
│  ├─ 修改日限额（投放中可调）
│  ├─ 暂停/恢复投放
│  └─ 添加新创意或替换
└─ 数据分析
   ├─ 按时间查看曝光、点击、转化
   ├─ 按位置分析效果
   ├─ 按创意对比
   └─ 计算 ROI

【第五步】数据统计
├─ 实时看板
│  ├─ 今日曝光、点击、花费
│  ├─ 各位置效果对比
│  └─ CTR 趋势
├─ 历史数据
│  ├─ 自定义时间范围
│  ├─ 导出报表
│  └─ 趋势分析
└─ 优化建议
   ├─ 识别低效位置
   ├─ 推荐创意优化
   └─ 预测 ROI
```

## 迁移策略

### 从旧系统迁移到新系统

**步骤 1：创建分类**
```sql
INSERT INTO ad_position_categories (code, name, sort_order) VALUES
('home', '首页', 10),
('channel', '频道页', 20),
('detail', '详情页', 30),
('list', '列表页', 40);
```

**步骤 2：迁移广告位**
```sql
-- 将 AdPosition 数据转换
INSERT INTO ad_positions (category_id, name, description, scene, max_ads, status)
SELECT cat.id, ap.name, '', ap.scene, 5, 'enabled'
FROM ad_positions ap
JOIN ad_position_categories cat ON cat.code = 'home';
```

**步骤 3：迁移广告**
```sql
-- 将 Ad 转换为 AdCampaign + AdCreative
-- 每个 Ad 创建一个 Campaign 和 Creative
INSERT INTO ad_campaigns (name, start_date, end_date, budget, pricing_model, status)
SELECT CONCAT('迁移-', title), CURDATE(), DATE_ADD(CURDATE(), INTERVAL 30 DAY), 
       0, 'cpm', 'active'
FROM ads;

INSERT INTO ad_creatives (campaign_id, title, image_url, link_type, link_value, status)
SELECT ac.id, a.title, a.image, a.link_type, a.link_value, 'approved'
FROM ads a
JOIN ad_campaigns ac ON ac.name = CONCAT('迁移-', a.title);
```

**步骤 4：保留兼容性**
- 保留旧的 AdPosition 和 Ad 表用于查询
- 用视图（VIEW）将新表数据映射到旧表结构
- 逐步迁移前端调用，最后下线旧 API

通过此次重构，广告管理系统将：

✅ **逻辑清晰**：广告位 → 计划 → 创意 → 投放 → 数据的完整链条
